# 検索・タグ・業種分類 設計

## 基本方針

- 検索 UI は公開一覧の体験を基準に `q / zone / status / tag / sort / density` を正式採用する
- タグは本人入力項目ではなく、管理者レビューを通す派生情報として扱う
- `admin/tags` の主目的は `未タグ会員の割当キュー` であり、タグ辞書やルールの直接編集ではない
- `gas-prototype/` のタグ辞書は初期カテゴリ検討の参考に使うが、本番バックエンド仕様の正本にはしない
- 実装先は `apps/web` の一覧・検索 UI と `apps/api` のタグ付与/同期処理で分離する

---

## 公開一覧の検索 UI

### 検索パラメータ

| key | 用途 | 初期値 |
|-----|------|--------|
| `q` | キーワード検索 | 空文字 |
| `zone` | UBM区画 | `all` |
| `status` | 参加ステータス | `all` |
| `tag` | タグ複数選択 | 空配列 |
| `sort` | 並び替え | `recent` |
| `density` | 表示密度 | `comfy` |

### 画面上の並び

1. キーワード検索
2. UBM区画
3. 参加ステータス
4. 並び替え
5. クリア
6. タグ絞り込み
7. 件数表示
8. 一覧本体

### 表示密度

| 値 | 用途 |
|----|------|
| `comfy` | 主要情報 + 概要2行の標準表示 |
| `dense` | 省スペースで多件数を見たい場合 |
| `list` | テーブル的に比較したい場合 |

## 検索対象

- `fullName`
- `nickname`
- `occupation`
- `location`
- `businessOverview`
- `skills`
- `canProvide`
- `selfIntroduction`
- `tags`

### 検索対象から外すもの

- 会員限定項目
- 管理用項目
- 削除済みメンバー
- 非公開メンバー

---

## 並び替え

| 値 | 意味 |
|----|------|
| `recent` | 最終更新順 |
| `name` | 氏名順 |

MVP では上記2種類に限定する。

## タグ設計

### 正式な扱い

- タグは検索補助と一覧理解を早めるための派生情報
- メンバー本人はタグを編集しない
- 未タグ会員は `admin/tags` のキューで管理者が順に処理する
- タグ割当後に一覧と詳細へ反映する

### 管理画面での扱い

- 左側に `未タグ会員キュー`
- 右側に `会員要約 / 事業概要 / スキル / タグ選択`
- 保存したら次の会員へ進める

### タグ辞書

- `gas-prototype/` にある `6カテゴリ30タグ` は初期候補として参照してよい
- ただし本 spec の主眼は辞書編集ではなく `割当レビュー` に置く

---

## 空状態

### 一覧検索

- 条件に合う会員がいない場合は空状態を表示する
- `絞り込みをクリア` を必ず表示する

### タグ割当

- 未タグ会員がゼロなら `未タグ付けのメンバーはいません` を表示する

---

## 不採用

- 公開一覧に `公開状態` フィルターを出す構成
- 本人がタグを追加・削除する構成
- `admin/tags` をタグルール編集の主画面にする構成
- 固定カラムだけで業種分類を表現する構成
