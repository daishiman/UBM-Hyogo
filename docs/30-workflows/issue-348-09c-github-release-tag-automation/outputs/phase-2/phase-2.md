## template 採用根拠（なぜ markdown placeholder か）

| 候補 | 採否 | 理由 |
| --- | --- | --- |
| **markdown + `{{PLACEHOLDER}}`** | **採用** | shell の `sed` / `envsubst` 相当で決定論的に展開可能。GitHub Release は markdown render するためそのまま投入できる |
| Mustache / Handlebars | 不採用 | 追加 runtime 依存（node / 別 CLI）。`gh` のみで完結する設計と齟齬 |
| heredoc 直書き | 不採用 | template と生成スクリプトが分離できず、レビュー粒度が悪化 |

## placeholder 一覧

| placeholder | 型 | 説明 | 必須 |
| --- | --- | --- | --- |
| `{{TAG}}` | string | release tag (`vYYYYMMDD-HHMM`) | yes |
| `{{COMMIT}}` | string (40 hex) | target commit full sha | yes |
| `{{COMMIT_SHORT}}` | string (7-12 hex) | 表示用の short sha | yes |
| `{{RELEASE_DATE_JST}}` | string (`YYYY-MM-DD HH:MM JST`) | tag に対応する日時の人間可読表記 | yes |
| `{{CHANGELOG_URL}}` | URL | Phase 12 `documentation-changelog.md` の GitHub URL | yes |
| `{{RUNTIME_EVIDENCE_URL}}` | URL | Phase 11 evidence ディレクトリの GitHub URL | yes |
| `{{ROLLBACK_EVIDENCE_URL}}` | URL or `N/A` | Phase 11 rollback evidence URL（無ければ `N/A`） | yes |
| `{{KNOWN_FOLLOW_UP}}` | markdown list | unassigned-task / open issue の箇条書き | yes（空なら `- なし`） |
| `{{PRODUCTION_WEB_URL}}` | URL | production web URL（runbook 既知値） | yes |
| `{{PRODUCTION_API_URL}}` | URL | production api URL（runbook 既知値） | yes |

## template 構造（セクション仕様）

```markdown
# Release {{TAG}}

- **Commit**: `{{COMMIT_SHORT}}` ({{COMMIT}})
- **Released at**: {{RELEASE_DATE_JST}}
- **Production**: {{PRODUCTION_WEB_URL}} / {{PRODUCTION_API_URL}}

## Changelog
- Source of truth: {{CHANGELOG_URL}}

## Runtime evidence (Phase 11)
- {{RUNTIME_EVIDENCE_URL}}

## Rollback evidence
- {{ROLLBACK_EVIDENCE_URL}}

## Known follow-up
{{KNOWN_FOLLOW_UP}}

---
Generated by `scripts/release/generate-release-notes.sh`. Do not edit manually.
```

## セクション要件

| セクション | 要件 |
| --- | --- |
| H1 | `Release {{TAG}}` 固定。tag は immutable のため見出し変更不可 |
| Metadata 行 | commit short / full / 日時 / production URL を 1 ブロックに集約 |
| Changelog | URL のみ。本文は Phase 12 を SSOT とし重複転記しない |
| Runtime evidence | URL のみ。Phase 11 ディレクトリへのリンク |
| Rollback evidence | 無ければ `N/A` を明示（空欄禁止） |
| Known follow-up | 空のとき `- なし` を出力（空文字禁止） |
| Footer | 自動生成記号として固定文字列を末尾に置き、手動編集禁止を明示 |

## 不変条件

- 同一入力 → 同一出力（決定論性）
- placeholder 未解決の `{{...}}` が 1 つでも残った場合は exit 1（Phase 3 で強制）
- template 自体には PII / Secret を埋め込まない
