## Part 1: 中学生レベルの説明

このタスクは、Webサイトを公開する前に「大事な画面がちゃんと動くか」を自動で見回る準備をする作業です。

たとえば文化祭の受付を考えると、当日に人が来てから「入口の案内板がない」「名簿が開けない」と気づくと困ります。そこで前日に、入口、受付、控室、案内板を順番に歩いて確認するチェック係を用意します。Playwright は、このチェック係をブラウザの中で動かす道具です。

今回作ったものは、まだ「前日の見回り結果の写真」ではありません。どの入口を見に行くか、どの順番で見るか、写真をどのフォルダへ置くかを決めた地図とチェック表です。実際の見回りは、上流の画面実装がそろった後、後続タスクまたは 09a staging smoke で行います。

### Error Handling And Edge Cases

| Case | Current handling | Full-execution requirement |
| --- | --- | --- |
| All specs skipped | Accepted only in `scaffolding-only` | Fail if used as CI gate |
| Auth fixture returns placeholder token | Documented gap | Generate Auth.js-compatible session or log in through UI |
| D1 seed/reset placeholder | Documented gap | Deterministic local/staging seed before tests |
| PNG count `< 30` | Current state, not PASS | Fail AC-7 |
| axe report placeholder | Current state, not PASS | Fail AC-8 if critical / serious violations exist |
