## Step 2: `scripts/regenerate-static-manifest.mjs` 実装

### 変更対象

- 新規: `scripts/regenerate-static-manifest.mjs`

### 関数シグネチャ

```js
// scripts/regenerate-static-manifest.mjs
import { readFile, writeFile } from "node:fs/promises";
import { createHash } from "node:crypto";

/**
 * Source spec markdown から canonical static-manifest を決定論的に生成する。
 * @param {Object} opts
 * @param {string} [opts.sourceSpecPath] - default: docs/00-getting-started-manual/specs/01-api-schema.md
 * @param {string} [opts.outputPath]     - default: apps/api/src/repository/_shared/generated/static-manifest.json
 * @param {() => Date} [opts.clock]       - test 用注入。本実装は使用しない（決定論性のため）
 * @returns {Promise<{ ok: true, manifestPath: string, sourceSpecHash: string }>}
 */
export async function regenerateStaticManifest(opts = {}) { /* ... */ }
```

### 期待挙動

1. `sourceSpecPath` を読み、`\r\n` を `\n` に正規化、末尾改行 1 個に丸めて canonicalize
2. `sha256(canonicalized)` を計算 → `sourceSpecHash`
3. source spec の git log 最終 commit ISO timestamp と short SHA を `child_process.execFileSync("git", ["log", "-1", "--format=%H %cI", "--", sourceSpecPath])` で取得（git 不在時は固定文字列 `"deterministic"` を fallback）
4. markdown の section 表 / field 表を parse して `sections` / `fields` を構築
5. JSON を **キー順固定**（Phase 2 で確定した順序）で `JSON.stringify(obj, null, 2) + "\n"` で書き出す
6. `Object.fromEntries(Object.entries(fields).sort(([a], [b]) => a.localeCompare(b)))` で fields キーを辞書順に固定
7. `sections` は `position` 昇順で sort

### 副作用

- 出力ファイル 1 件への上書きのみ。ネットワーク I/O / D1 / Cloudflare API 呼び出しなし

## Step 3: `scripts/verify-static-manifest.mjs` 実装

### 変更対象

- 新規: `scripts/verify-static-manifest.mjs`

### 関数シグネチャ

```js
/**
 * @param {Object} opts
 * @param {string} [opts.manifestPath]
 * @param {string} [opts.sourceSpecPath]
 * @returns {Promise<
 *   | { ok: true }
 *   | { ok: false; reason: "sourceSpecHashDrift"; expected: string; actual: string }
 *   | { ok: false; reason: "missingSourceSpec"; path: string }
 *   | { ok: false; reason: "invalidSchema"; details: unknown }
 * >}
 */
export async function verifyStaticManifest(opts = {}) { /* ... */ }
```

### 期待挙動

1. manifest を JSON.parse、必須キー `sourceSpecHash` 等が無ければ `{ ok: false, reason: "invalidSchema", details }`
2. `sourceSpecPath` 不在なら `{ ok: false, reason: "missingSourceSpec", path }`
3. canonicalize（Step 2 と同一ロジック）して sha256 計算
4. `manifest.sourceSpecHash` と一致なら `{ ok: true }`、不一致なら `{ ok: false, reason: "sourceSpecHashDrift", expected, actual }`

### CLI entry

```js
// scripts/verify-static-manifest.mjs 末尾
if (import.meta.url === `file://${process.argv[1]}`) {
  const result = await verifyStaticManifest();
  if (!result.ok) {
    process.stderr.write(`[verify-static-manifest] FAIL reason=${result.reason}\n`);
    process.stderr.write(JSON.stringify(result, null, 2) + "\n");
    process.exit(1);
  }
  process.stdout.write("[verify-static-manifest] OK\n");
}
```

## Step 5: `metadata.ts` resolver に sourceSpecHash 検証フック追加

### 変更対象

- 編集: `apps/api/src/repository/_shared/metadata.ts`

### 内容

`MetadataResolver` factory で manifest を読み込む際、import 時の hash 整合性を検証して drift 時 `logWarn` する（実行時 fail はしない・観測のみ）。retirement 条件への参照コメントを関数 doc に追加。

```ts
// metadata.ts の上部
/**
 * Static manifest baseline.
 * Retirement: see docs/00-getting-started-manual/specs/01-api-schema.md § Static Manifest Retirement Condition
 * Stale detection: scripts/verify-static-manifest.mjs (CI gate: pnpm verify:static-manifest)
 */
import staticManifest from "./generated/static-manifest.json" assert { type: "json" };
import { logWarn } from "../../lib/logger";

if (!staticManifest.sourceSpecHash || !staticManifest.sourceSpecHash.startsWith("sha256:")) {
  logWarn({
    code: "UBM-MANIFEST-INVALID-SCHEMA",
    note: "static-manifest.json is missing sourceSpecHash. Run pnpm regenerate:static-manifest.",
  });
}
```
