# シークレット衛生

## リポジトリ層のシークレット扱い

本タスク（02a）ではシークレットを新たに導入しない。

## D1 バインディングの扱い

- D1 へのアクセスは `DB` バインディング名で統一（wrangler.toml 定義済み）
- `DB` バインディングは Cloudflare Workers のランタイム環境で自動注入される
- テストコードでは `MockD1` を使用し、実際のバインディングには依存しない

## 設計上の境界

- `DbCtx` interface は `D1Db` のみを含み、シークレットや認証情報は持たない
- リポジトリ関数は `DbCtx` のみを受け取り、他の環境変数への依存なし

## チェックリスト

- [x] `.env` ファイルにシークレットを記録しない
- [x] テストコードに実際の DB 認証情報を含まない
- [x] artifacts.json にシークレットを記録しない
- [x] 新規シークレットなし
