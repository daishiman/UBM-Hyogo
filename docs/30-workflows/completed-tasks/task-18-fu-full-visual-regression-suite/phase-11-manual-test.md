# Phase 11: 手動テスト（VISUAL）

## 目的

VISUAL classification として 51 screenshot baseline を evidence 化する。CI artifact retention の確認、ローカル差分確認手順を提示する。

---

## 入力

- baseline update workflow が merge した PR の baseline 一式
- `apps/web/playwright/tests/visual-full/full-visual.spec.ts-snapshots/` 配下 51 png

---

## 1. VISUAL evidence 配置

| パス | 内容 |
|------|------|
| `outputs/phase-11/evidence/baseline/INDEX.md` | 51 baseline の一覧（slug / viewport / リンク） |
| `outputs/phase-11/evidence/baseline/README.md` | 取得手順 + コミット情報 |

> 51 png をワークフロー output 配下に重複コピーすると差分管理が二重化するため、**evidence は INDEX.md のリンク参照のみ**とし、png 実体は `apps/web/playwright/tests/visual-full/full-visual.spec.ts-snapshots/` に集約する。

## 3. CI artifact retention

`.github/workflows/playwright-visual-full.yml` の `actions/upload-artifact@v4` 設定:

| 項目 | 値 |
|------|-----|
| artifact name | `visual-full-{viewport}-diff` |
| path | `apps/web/test-results/` |
| retention-days | `14` |
| upload condition | `if: failure()` |

→ diff 発生時のみ artifact が残り、14 日後に自動削除されることを Phase 9 CI dry-run で確認済み。

## 4. ローカル差分確認手順

```bash
# 1. リポジトリ最新化
git pull origin dev

# 2. baseline 再取得（macOS では確認のみ・commit 不可）
mise exec -- pnpm --filter @ubm-hyogo/web build
mise exec -- pnpm --filter @ubm-hyogo/web test:visual-full

# 3. 差分発生時の HTML report
mise exec -- pnpm --filter @ubm-hyogo/web exec playwright show-report
```
