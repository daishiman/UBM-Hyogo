## 主要な型・関数シグネチャ（CONST_005 / Phase 2 から再掲）

```ts
// scripts/postmortem/generate-postmortem.ts

export type PostmortemInput = {
  release: string;          // "vX.Y.Z"
  commit: string;           // "[0-9a-f]{7,40}"
  evidencePath: string;     // 09c Phase 11 evidence directory（必須・実在チェック）
  rollbackEvidencePath: string; // rollback 実施記録 path
  occurredAt: string;       // ISO8601（S4: 冪等性のため明示入力）
  detectedAt?: string;
  resolvedAt?: string;
  severity?: string;
};

export type ValidationResult =
  | { ok: true; input: PostmortemInput }
  | { ok: false; reason: string };

export function validateInput(raw: Record<string, string | undefined>): ValidationResult;
// 副作用なし。release / commit の正規表現チェック、必須引数欠落チェック。

export function ensureEvidencePathExists(path: string): { ok: boolean; reason?: string };
// 副作用: fs.statSync で実在チェックのみ（read しない）。
// directory 確認 + 配下に main.md 存在確認の二段。

export function generatePostmortem(input: PostmortemInput, template: string): string;
// 副作用なし。入力 + template文字列 → markdown 文字列。決定論的。Date.now() / Math.random() を使わない。
export function loadTemplate(): string;
// 副作用: template.md の read のみ。CLI/loader 層に閉じる。
// blame 表現を含む見出し / placeholder を出力に書き込まない（S1）。

// CLI 層
async function main(argv: string[]): Promise<number>;
// 副作用: stdout / stderr 書き込み、`--out` 指定時のみ fs.writeFileSync。
// exit code: 0 / 1 / 2（Phase 1 終了コード表に従う）。
```

CLI parser は **`node:util` の `parseArgs`** を使う（外部依存追加禁止 / S5）。

## markdown template の固定見出しサンプル全文（S1）

`docs/30-workflows/runbooks/postmortem/template.md` にそのまま貼り付ける本文を以下に固定する。`{{...}}` placeholder は `generatePostmortem` が単純な文字列置換で埋める（テンプレートエンジン依存追加なし）。

````markdown
# Postmortem: {{release}}

> Generated by `pnpm postmortem:generate`.
> Source of truth for evidence: 09c Phase 11 evidence (link below).

## Header

| 項目 | 値 |
| --- | --- |
| Release | `{{release}}` |
| Commit | `{{commit}}` |
| Occurred at | `{{occurredAt}}` |
| Detected at | `{{detectedAt}}` |
| Resolved at | `{{resolvedAt}}` |
| Severity | `{{severity}}` |
| Evidence path | `{{evidencePath}}` |
| Rollback evidence | `{{rollbackEvidencePath}}` |

## Timeline

| Time (UTC) | Event |
| --- | --- |
| {{occurredAt}} | (記入) |
|  |  |
|  |  |

> Use events and actions as subjects. Do not use person names as the subject.

## Impact

- Affected scope: (記入)
- Affected users: (記入)
- Affected features: (記入)

## Detection

- Signal source: (dashboard / alert / user report)
- First detected at: `{{detectedAt}}`
- Lag from occurrence: (記入)

## Response

- Action taken: (記入)
- Rollback type: (worker / pages / D1 / cron / none)
- Rollback evidence: `{{rollbackEvidencePath}}`

## Root Cause

> Subject must be **code / configuration / process**, not a person.

- Technical cause: (記入)
- Contributing factors: (記入)

## Prevention

- Monitoring: (追加するアラート / dashboard)
- Test: (追加する unit / integration / E2E)
- Runbook update: (更新対象 runbook の relative path)

## Follow-up Issues

> 起票手順は `docs/30-workflows/runbooks/postmortem/README.md` を参照。
> 1 postmortem あたり 1 件以上必須（Prevention セクションが空のままなら未完）。

- [ ] `[postmortem-followup] (概要)` — tracking: (issue link)
- [ ] `[postmortem-followup] (概要)` — tracking: (issue link)
````

このサンプルが**正本**。スクリプトは **template.md を read** して `{{...}}` を `PostmortemInput` の値で置換し、未指定の optional は空文字列 `""` で置換する（`undefined` を出力しない・S4 冪等性確保）。

### ステップ 4: スクリプト骨格作成（pure 関数層）

`scripts/postmortem/generate-postmortem.ts` を新規作成。以下の順で実装する（TDD: Phase 4 TC を先に Red にしてから Green にする）。

1. `PostmortemInput` / `ValidationResult` 型定義
2. `validateInput(raw)` 実装
   - release: `/^v\d+\.\d+\.\d+/` で正規表現マッチ。fail で `ok:false`
   - commit: `/^[0-9a-f]{7,40}$/` で正規表現マッチ
   - 必須欠落（`evidencePath` `rollbackEvidencePath` `occurredAt`）で `ok:false`
3. `ensureEvidencePathExists(path)` 実装
   - `fs.statSync` で `ENOENT` → `{ ok:false, reason:"evidence path not found: <path>" }`
   - `isDirectory()` false → `{ ok:false }`
   - `path.join(path, 'main.md')` の statSync で `ENOENT` → `{ ok:false }`
4. `loadTemplate()` 実装
   - `template.md` を `fs.readFileSync` で読む（**path は固定**: `docs/30-workflows/runbooks/postmortem/template.md` の repo-root 相対）
5. `generatePostmortem(input, template)` 実装
   - placeholder `{{release}}` `{{commit}}` `{{evidencePath}}` `{{rollbackEvidencePath}}` `{{occurredAt}}` `{{detectedAt}}` `{{resolvedAt}}` `{{severity}}` を `String.prototype.replaceAll` で置換
   - optional 値 (`detectedAt` / `resolvedAt` / `severity`) が undefined のときは空文字列 `""` で置換（S4: 出力決定論性）
   - **`Date.now()` / `Math.random()` / `process.env.HOSTNAME` を使わない**

> note: `generatePostmortem` は template 文字列を受け取る pure 関数とし、fs read は `loadTemplate()` / CLI 層に分離する。

### ステップ 5: CLI 層 `main(argv)` 実装

1. `node:util` の `parseArgs` で argv パース
2. `validateInput` → fail なら stderr 出力 + `return 1`
3. `ensureEvidencePathExists` → fail なら stderr 出力 + `return 1`
4. `generatePostmortem(input, loadTemplate())` を呼ぶ
5. `--out` 指定: `fs.writeFileSync(out, markdown)`。失敗で `return 2`
6. `--out` 未指定: `process.stdout.write(markdown)`
7. `return 0`
8. ファイル末尾で `if (import.meta.url === ...)` 形式の entry point 判定 → `main(process.argv.slice(2)).then(code => process.exit(code))`
