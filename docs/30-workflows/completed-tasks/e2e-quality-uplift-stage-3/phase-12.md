# Phase 12: 中学生レベル概念説明

## このタスクが何をやっているのか

ウェブサイト（UBM 兵庫支部会の会員サイト）を作るとき、コードを書いたあと「ちゃんと動くか」を **自動でテストするロボット**を仕込みます。テストロボットは大きく 3 種類あります。

1. **E2E（イー・ツー・イー）テスト**: 実際にブラウザを動かして「ボタン押したらちゃんと画面が出るか」を確認するロボット
2. **Lighthouse（ライトハウス）テスト**: ページが速いか、見やすいかを点数で測るロボット
3. **Coverage（カバレッジ）テスト**: コードのうち何 % にテストが書いてあるかを数えるロボット

今までこの 3 つのロボットは「動くようにはなっていた」のですが、「テストに落ちても、コードを本番に合体させていい」状態でした（鍵がかかっていないドアみたいなもの）。

このタスクでは、その 3 つのロボットを **「テストに落ちたらコード合体禁止！」のロックに昇格** させます。これを「ブランチプロテクション」と呼びます。

## なぜ必要なのか

- せっかくテストロボットを作っても、合格しなくても合体できるなら意味がない
- バグや遅いページが本番に出てしまうと、利用者が困る
- 「テスト OK じゃないと PR がマージできない」にすると、毎回ちゃんとチェックされる

## 何をどう変えたのか

### 変更前
- テストロボット（e2e、lighthouse）は動いているが、ロックされていない
- 「合体禁止リスト」には 3 つしか入っていなかった: `ci`, `Validate Build`, `coverage-gate`

### 変更後
- 「合体禁止リスト」に **2 つ追加** する
  - `e2e-tests-coverage-gate`: 3種類のブラウザ E2E と coverage をまとめて確認するチェック
  - `lighthouse-ci`: 速度・見やすさチェック
- dev / main の両方で同じ 5 チェックに揃える

### おまけの修正

Lighthouse ロボットを動かすとき、「サーバーが起動するのを待つ」処理が手作りループでちょっと不安定だったので、`wait-on` という専用ツールに置き換えて安定させました。

## ファイル単位での変更まとめ

| ファイル | 何が変わったか |
|---------|--------------|
| `.github/branch-protection/dev.json`（新規） | 「dev ブランチの鍵リスト」を JSON ファイルで管理 |
| `.github/branch-protection/main.json`（新規） | 「main ブランチの鍵リスト」 |
| `.github/branch-protection/apply.sh`（新規） | 鍵リストを GitHub に適用するスクリプト |
| `scripts/verify-branch-protection.sh`（新規） | 鍵リストが正しいか毎回チェックするスクリプト |
| `.github/workflows/lighthouse.yml`（編集） | サーバー起動待ちを wait-on で安定化 |

## 「PR ブロッキング」って何

GitHub の「Pull Request（PR）」は、書いたコードをみんなのコード（dev / main）に**合体させる申請**のこと。

「PR ブロッキング」= テストロボットが OK 出さない限り、合体ボタン（merge）が押せなくなる仕組み。

今回追加するのはまさにこれです。

## 不安要素と対策

| 心配 | 対策 |
|------|------|
| 急にロックがかかって、自分の作業中の PR が merge できなくなったらどう？ | このタスクを完了してロック ON にする前に、現状の PR が全部テスト OK になっているか確認してから apply する |
| ロボット名がちょっと変わるとロックが空振りしない？ | 名前は `e2e-tests.yml` 内の `name:` フィールドと完全一致させる。1 文字でもズレたらアウトなので、apply 後に必ず drift 検査スクリプトで OK 確認する |
| 元に戻したくなったら？ | 適用前に「Before スナップショット」を JSON で保存しておくので、それを使えば元の状態に戻せる |

## まとめ

「動くだけだったテストロボット」を「動かないとマージできないロボット」に昇格させる。それだけのタスクです。コードの動きは変わりません。**安全装置を ON にする**作業です。
